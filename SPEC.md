### 4.2 房间数据结构

```go
type Room struct {
    ID       string    // 房间唯一ID (6位字母)
    Code     string    // 分享码 (6位数字)
    HostID   string    // 房主 PlayerID
    Status   string    // waiting | playing | settling

    Players  []*Player // 按座位顺序
    MaxSeats int        // 最多9人

    CreatedAt time.Time
}

type Player struct {
    ID       string    // 玩家唯一ID (UUID)
    Name     string    // 显示名称
    Seat     int       // 座位号 0-8
    Chips    int64     // 当前筹码
    Connected bool     // 是否在线
}
```

## 五、德州扑克核心逻辑

### 5.1 游戏流程（简化版）

```
┌─────────┐
│  开始    │
└────┬────┘
     │ 发牌 (每人2张)
     ▼
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 翻牌前  │───▶│ 翻牌    │───▶│ 转牌    │───▶│ 河牌   │
│(Pre-flop)│    │(Flop)   │    │(Turn)   │    │(River)  │
└────┬────┘    └────┬────┘    └────┬────┘    └────┬────┘
     │               │               │               │
     ▼               ▼               ▼               ▼
  动作轮次      动作轮次        动作轮次        动作轮次
  (Betting)    (Betting)       (Betting)       (Betting)
     │               │               │               │
     └───────────────┴───────────────┴───────────────┘
                                             │
                                             ▼
                                        ┌─────────┐
                                        │  结算   │
                                        └─────────┘
```

### 5.2 动作类型

| 动作 | 说明 | 触发条件 |
|------|------|----------|
| 过牌 (Check) | 不下注，转给下家 | 无人加注时 |
| 跟注 (Call) | 下与当前最高注相同的注 | 有人加注时 |
| 加注 (Raise) | 超过当前最高注 | 筹码足够 |
| 弃牌 (Fold) | 退出本轮 |
| All-in | 押上全部筹码 | 筹码不足跟注时 |

### 5.3 牌型大小（从大到小）

| 牌型 | 说明 | 示例 |
|------|------|------|
| 皇家同花顺 | 同花顺最大 | A♠K♠Q♠J♠10♠ |
| 同花顺 | 同花+顺子 | 9♥8♥7♥6♥5♥ |
| 四条 | 四张同点 | A♠A♥A♦A♣K♠ |
| 葫芦 | 三条+对子 | A♠A♥A♦K♣K♠ |
| 同花 | 同花色 | A♠J♠8♠6♠2♠ |
| 顺子 | 连续点数 | 9♠8♥7♦6♣5♠ |
| 三条 | 三张同点 | A♠A♥A♦K♣Q♠ |
| 两对 | 两个对子 | A♠A♥K♣K♠Q♠ |
| 一对 | 一个对子 | A♠A♥K♣Q♠J♠ |
| 高牌 | 单张最大 | A♠J♥8♦6♣3♠ |

### 5.4 牌型判断算法

```
1. 统计每种点数出现次数 → [4,2,1] = 四条+对子 = 葫芦
2. 检查是否为同花（5张同花色）
3. 检查是否为顺子（连续5张）
4. 组合判断：
   - 同花 + 顺子 = 同花顺 / 皇家同花顺
   - 四条 = 4个相同点数
   - 葫芦 = 3个+2个
   - 同花 = 同花色但非顺子
   - 顺子 = 连续但非同花
   - 三条 = 3个相同
   - 两对 = 2个+2个
   - 一对 = 2个相同
   - 高牌 = 其他
```

## 六、通信协议设计

### 6.1 WebSocket 消息格式

```json
{
  "type": "action_type",
  "data": {},
  "seq": 123
}
```

### 6.2 消息类型定义

| 方向 | type | 说明 |
|------|------|------|
| C→S | join_room | 加入房间 |
| C→S | leave_room | 离开房间 |
| C→S | start_game | 开始游戏（房主） |
| C→S | player_action | 玩家动作 |
| S→C | room_update | 房间状态更新 |
| S→C | game_state | 游戏状态推送 |
| S→C | game_result | 结算结果 |
| S→C | error | 错误信息 |

### 6.3 详细消息定义

**玩家动作 (player_action)**
```json
{
  "type": "player_action",
  "data": {
    "action": "call|raise|fold|check|allin",
    "amount": 100  // 加注金额（raise时必填）
  }
}
```

**游戏状态 (game_state)**
```json
{
  "type": "game_state",
  "data": {
    "phase": "preflop|flop|turn|river|showdown",
    "pot": 500,
    "communityCards": ["Ah", "Kd", "7s"],
    "currentPlayer": 2,
    "players": [
      {"id": "xxx", "name": "A", "cards": ["As", "Ac"], "chips": 1000, "status": "active"},
      {"id": "yyy", "name": "B", "cards": [], "chips": 800, "status": "fold"}
    ],
    "bets": [100, 100, 0, 0],  // 每位玩家本轮下注
    "toCall": 100  // 跟注金额
  }
}
```
